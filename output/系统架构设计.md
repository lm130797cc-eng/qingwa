## 1. 设计目标 (Architectural Goals)

本系统旨在构建一个轻量级、模块化、高可扩展的预测合成平台。遵循蚂蚁军团架构标准，确保每一行代码都符合八卦哲学的平衡与秩序。

*   **极简主义 (Minimalism)**: 纯 PowerShell 实现，无 Python 依赖。
*   **模块化 (Modularity)**: 各组件松耦合，通过标准化接口通信。
*   **二进制八卦 (Binary-Bagua)**: 核心逻辑采用 0/1 序列与五行流转。
*   **安全合规 (Security)**: 内置安全审计，杜绝危险操作。

## 2. 模块划分 (Module Decomposition)

系统划分为五大逻辑层级，对应五行属性：

### 2.1 输入层 (Input Layer) - [金]
*   **职责**: 接收外部指令与原始数据。
*   **组件**: Input_Receiver.ps1
*   **数据格式**: [分子, 分母] (例如: [8, 9])

### 2.2 运算层 (Calculation Layer) - [火]
*   **职责**: 执行核心预测逻辑，计算八卦序列与五行流转。
*   **组件**: PyramidCore.ps1 (三轴逻辑引擎)
*   **核心算法**:
    *   **Bagua Sequence**: 将数字转换为二进制序列。
    *   **Wuxing Flow**: 计算动态平衡 (e.g., MU -> HUO)。

### 2.3 扩展层 (Extension Layer) - [木]
*   **职责**: 动态加载特殊规则与高概率锁定逻辑。
*   **组件**: Plugin_Loader.ps1
*   **机制**: 监控 SpecialRules/ 目录，自动加载 .ps1 插件 (如 Rule_08_09_Lock.ps1)。

### 2.4 数据库层 (Database Layer) - [土]
*   **职责**: 存储历史记录与 G-Gas 余额。
*   **组件**: Data_Store.json (轻量级文件存储)
*   **字段**: HistoryRecords, GasBalance, AuditLogs.

### 2.5 UI 层 (UI Layer) - [水]
*   **职责**: 数据可视化与交互展示。
*   **组件**: UI_Builder.ps1 生成的 Dashboard.html / Console_View.ps1
*   **特性**: 极简界面，仅展示关键指标 (Current, Next, Confidence)。

## 3. 接口定义 (Interface Definitions)

模块间通信采用标准化 JSON 对象。

### 3.1 预测请求 (Prediction Request)
`json
{
    "type": "PREDICT",
    "payload": {
        "numerator": 8,
        "denominator": 9,
        "timestamp": "2026-02-07T12:00:00Z"
    }
}
`

### 3.2 预测响应 (Prediction Response)
`json
{
    "status": "SUCCESS",
    "result": {
        "current_pattern": "P",
        "next_prediction": "blue",
        "confidence": 0.85,
        "wuxing_flow": "MU->HUO"
    },
    "metadata": {
        "engine_version": "1.0.0",
        "applied_rules": ["Base_Logic", "Rule_08_09_Lock"]
    }
}
`

## 6. 部署与升级 (Deployment & Upgrade)

*   **部署**: 最终交付物为单文件 .ps1 或 .html，实现即插即用。
*   **升级**: 将新规则脚本放入 SpecialRules/ 即可生效，无需重启核心引擎。
